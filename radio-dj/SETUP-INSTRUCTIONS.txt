RADIO NOW — SIMPLE SETUP
========================

THE BIG PICTURE (3 pieces)
--------------------------
  1. Google Form     — artists submit songs + upload big WAV/MP3/cover files
  2. Google Sheet    — Form Responses 1 stores every submission (your database)
  3. Static website  — reads data/songs.json (refreshed from the sheet every 5 min)


IS SHEET1 THE SAME AS FORM RESPONSES 1?
---------------------------------------
  NO. They are different tabs in the same spreadsheet.

  • Form Responses 1 — auto-filled when someone submits the Google Form
  • Sheet1           — your older hand-built catalog (3 songs today)

  The website sync reads BOTH tabs and merges them (newest form row wins
  if the same artist + song appears twice).


THE SIMPLE FLOW
---------------
  Artist submits Google Form
       ↓
  Form Responses 1  (text + Drive links to MP3/WAV/Cover)
  Drive folders       (MP3 / WAV / Cover Art — set in the form)
       ↓
  GitHub sync every 5 minutes  →  data/songs.json  →  live website
       ↓
  Artist dashboard shows that artist's songs from songs.json
  (promo folders, downloads — matched by Artist Name)

---------------------
  ✓ Google Form linked to the sheet (you have this)
  ✓ Each file upload in the form pointed at the right Drive folder:
      MP3, WAV, Cover Art folders (see below)
  ✓ GitHub Action "Sync Google Sheet to JSON" enabled (runs every 5 min)
  ✓ Apps Script deployed for DJ accounts, ZIP downloads, audio streaming

  OPTIONAL Apps Script trigger (only if files land in the wrong folder):
    onArtistSongFormSubmit → On form submit
    Renames files to "Artist - Song Title.ext" and moves them.

ARTIST DASHBOARD
----------------
  "Submit a New Song" opens your Google Form in a new tab.
  No file upload on the website — Google Form handles large WAVs.

  After submit, the song appears on the site within ~5 minutes automatically.
  Artists do NOT run GitHub or touch the sheet.
